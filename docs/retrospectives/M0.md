### D3 — Frontend tooling + AST static check

**Scope:** Vite + **Tailwind 4** + TypeScript scaffold under `frontend/`;
django-vite wired into `config/settings/base.py` and `templates/base.html`;
Vite Compose service with host-install pattern (entrypoint script refuses
to start without `frontend/node_modules`); Nginx `/vite/` proxy with
runtime DNS resolution + WebSocket upgrade + friendly 503 fallback;
service-layer discipline AST check (advisory) at
`scripts/check_service_layer_discipline.py`; CI `service-discipline` and
`frontend-build` jobs.

**Verification:**
- `cd frontend && npm install --no-fund --no-audit` on host succeeds and
  produces `frontend/node_modules/`.
- `make frontend-typecheck` (host) — `tsc --noEmit` succeeds with strict mode.
- `make frontend-build` (host) — `vite build` produces `frontend/dist/.vite/manifest.json`.
- `make up` brings up Vite alongside web/worker/beat; Vite entrypoint
  refuses to start if `node_modules` is missing.
- `curl -sf http://mph.local/vite/@vite/client` returns HTTP 200.
- Manual HMR check: edit `frontend/src/styles/login_landing.css` while
  `make up` is running; browser refreshes the page automatically.
- `make lint-service-discipline` runs on host; emits warnings only and
  exits 0 (advisory in M0 per A.4.5).
- Tailwind browser CDN removed from `templates/base.html`; replaced by
  `{% vite_hmr_client %}` + `{% vite_asset 'src/entries/<entry>.ts' %}`.

**Decisions:**
- **Tailwind 4 (not 3) selected.** Guide H.1.3 says
  Tailwind 3.x; updated to Tailwind 4.x. Implementation uses
  `@tailwindcss/vite` plugin (no PostCSS pipeline, no autoprefixer —
  Lightning CSS handles both internally).
- **`@tailwindcss/container-queries` plugin dropped** — container queries
  are built into Tailwind 4. Functional intent of H.1.3 preserved.
- **`tailwind.config.cjs` retained** per H.2.1's "tokens defined ONCE"
  requirement, wired via `@config` directive in `_base.css`. v4 supports
  this dual mode (CSS `@theme` blocks OR JS config file).
- **Vite HMR routed through Nginx** (`/vite/` proxy with WebSocket upgrade
  + runtime DNS resolution) rather than the browser connecting directly to
  `:5173`. Keeps the dev experience on one hostname (`mph.local`).
- **Nginx uses Docker's embedded resolver (`127.0.0.11`) with per-request
  variable lookup** for both `vite` and `web` upstreams. This prevents
  Nginx from caching a "DNS failed" state at startup if an upstream is
  briefly unavailable.
- **Host-install pattern for `node_modules`** — npm install runs on the
  host (`make frontend-install` → `cd frontend && npm install`), not in
  the container. Required to make first-time setup tolerable on Windows
  + WSL2 where container-side bind-mount file I/O is effectively unusable.
  The Vite container's entrypoint refuses to start with empty
  `node_modules` and prints the recovery command.
- **Node 24 in container** matches host Node 24 to avoid native-module
  ABI mismatch warnings.
- **Service-discipline check stays advisory in M0**; promotion to blocking
  is scheduled for M2 per A.4.5. The check uses Python's stdlib `ast`
  module (no third-party dependency).
- **Email-template-preview bundle ships now** even though we don't yet
  have email-preview screens — Phase 1 invoice/notification HTML
  templates land in M5/M7 and we want the entry pre-wired.
