**Date:** 14-MAY-2026
**Scope:**

  services for the create path. `--reset` path retains direct ORM

**Verification:**

* `apps/platform/organizations/tests/services/test_create_organization.py`
  — 19 assertions:
  * Happy path produces org + 11 per-tenant roles with cloned
    capability sets.
  * Slug collision raises `OrganizationSlugInUseError` (NOT
    `IntegrityError`).
  * Slug collision rolls back the audit event along with the failed
    insert (single atomic boundary).
  * 9 invalid slug shapes parametrized as separate test cases.
  * 6 valid slug shapes parametrized as separate test cases.
  * Idempotency rule: the service does NOT no-op on existing slug.
  * Missing actor / nonexistent actor / invalid email all raise
    typed exceptions before any DB writes.
  * Role-clone failure inside the atomic block rolls back the
    Organization insert.
* `apps/platform/organizations/tests/services/test_assign_owner_membership.py`
  — 9 assertions:
  * Happy path creates Membership + MembershipRole with the correct
    owner role.
  * Both audit events (`MEMBERSHIP_CREATED`, `ROLE_ASSIGNED`) emit.
  * First org membership for a user is default; second is not.
  * Missing org / missing user / missing actor / duplicate membership
    all raise typed exceptions.
  * MembershipRole failure rolls back the Membership.
* `apps/platform/support/tests/test_seed_dev_tenant.py` — 7 assertions:
  * Existing 5 tests still pass after the service refactor.
  * New test asserts audit events emit through the service path.
* `make test` — full suite green.
* `make seed-dev` from a fresh `make clean` produces the working
  demo tenant; sign-in URL and credentials print at the end.

**Decisions:**

* **Audit-event registry deviations:** the guide's G.5.2 doesn't
  include `ORG_CREATED` or `MEMBERSHIP_CREATED` codes. We register
  both in `apps.platform.audit.services._KNOWN_EVENT_TYPES` now and
  amend G.5.2 in the M2 audit work. `ROLE_ASSIGNED` IS already in
  G.5.2.
* **`audit_emit` is a stub in M1.** It validates the transaction
  invariant (raises `AuditOutsideTransactionError` if called without
  an open transaction, per G.5.3) and records events in-memory when
  `MPH_AUDIT_RECORDING=True`. The M2 audit work replaces the recording
  buffer with a real `platform_audit.AuditEvent` insert. Service code
  calling the stub today won't need changes in M2.
* **`MPH_AUDIT_RECORDING` defaults `False` in base.py**; dev and test
  settings opt in. Production sees the stub's debug-level log line
  without buffer accumulation, so memory doesn't grow unbounded.
* **Slug collision routing:** `IntegrityError` with `"slug"` in the
  message becomes `OrganizationSlugInUseError`. Any other IntegrityError
  is re-raised unchanged so we don't accidentally swallow a real bug.
* **`create_organization` is strict, not idempotent.** The dev
  convenience layer (`seed_dev_tenant`) handles "already exists"
  before calling the service. Production callers should branch on
  `OrganizationSlugInUseError`.
* **`assign_owner_membership` sets `is_default_for_user=True` only if
  the user has no existing default membership.** This matches the M0
  D2 partial unique index that enforces "at most one default per user".
* **System User attribution for bootstrap calls:** `seed_dev_tenant`
  resolves the System User and passes its id as `actor_id` to both
  services, matching C.2's rule that system-triggered transitions
  attribute to the System User.

## Decisions Journal (M1-wide)

### M1 D2 — audit-event registry pending guide amendment

The audit stub in `apps.platform.audit.services` knows about two event
codes (`ORG_CREATED`, `MEMBERSHIP_CREATED`) that aren't in G.5.2 of the
guide. M2 audit work amends G.5.2 to include them; until then the
codes live in `_KNOWN_EVENT_TYPES` and emit through the stub.

### M1 D2 — service-layer pattern locked in

`create_organization` and `assign_owner_membership` set the pattern
every future service follows:

* `*` keyword-only arguments
* primitive types (UUID, str, decimal)
* single `transaction.atomic()` boundary
* `audit_emit(...)` inside the boundary
* typed exceptions for distinguishable failure modes
* no `request.user`; explicit `actor_id`

When we add tenant-domain services in M2+ (services for Lead, Quote,
Client, etc.) we replicate this shape exactly.
