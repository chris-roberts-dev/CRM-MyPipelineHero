## Sub-deliverables

### D1 — Tenancy primitives + isolation guardrail

**Date:** (filled on merge)
**Scope:** B.1.3 TenantOwnedModel abstract base; B.1.4 TenantManager +
TenantQuerySet; B.1.6 ensure_same_org helper + TenantViolationError /
OperatingScopeViolationError; B.2.6 resolve_location_ids_for_scopes
stub; B.1.7 tenant-isolation CI guardrail (BLOCKING); developer
primer docstring in `apps.common.tenancy.__init__`; placeholder
`apps/platform/organizations/services/__init__.py` for M1 D2's
`create_organization` to land into.

**Verification:**

- `apps/common/tenancy/tests/test_isolation_guardrail.py` — 9 assertions
  against the live model registry:
  - Every `is_tenant_owned=True` model uses TenantManager.
  - Every `is_tenant_owned=True` model has an `organization` ForeignKey.
  - Every such FK targets `platform_organizations.Organization`.
  - Every such FK uses `on_delete=PROTECT`.
  - Organization itself is NOT a TenantOwnedModel.
  - User is NOT a TenantOwnedModel.
  - Membership is NOT a TenantOwnedModel.
  - Capability and Role are NOT TenantOwnedModel.
  - TenantManager.use_in_migrations is False.
- `apps/common/tenancy/tests/test_utils.py` — 7 assertions on
  ensure_same_org behavior + stub guard on resolve_location_ids_for_scopes.
- `apps/common/tenancy/tests/test_manager_surface.py` — manager class
  invariants (use_in_migrations, base intersect no-op, for_org signature).
- `make test` — full suite green.
- `make lint` — ruff + mypy clean.
- `make migrate` — no new migrations expected (TenantOwnedModel is abstract).

**Confirms:**

- `Organization` (`apps.platform.organizations.models.Organization`)
  does NOT inherit from `TenantOwnedModel`. It IS the tenant root;
  inheritance would create a self-FK and break the model. Verified by
  `test_organization_is_not_tenant_owned`.

**Decisions:**

- **TenantManager does not auto-filter** queries. Service-layer code
  uses explicit `for_org` / `for_membership` calls. The alternative
  (auto-filter) would require every B.1.5 exception path to opt out,
  which inverts the safety posture. (Matches B.1.4 line 935.)
- **`resolve_location_ids_for_scopes` ships as a stub that raises
  NotImplementedError.** The Region/Market/Location models land later
  in M1; calling the stub today is a bug, so explicit failure beats
  silent wrong answers. The implementation lands with B.2.2.
- **`MembershipScopeAssignment` (B.2.4) deferred.** Region/Market/
  Location come first; the scope-assignment row references those.
  Scheduled for M1 D2 or D3.
- **`Membership` is NOT a `TenantOwnedModel` subclass** even though it
  has an organization FK. It's the JOIN row between User and
  Organization, and the platform console queries it across tenants.
  Membership uses `models.Manager` (the Django default). Documented in
  `apps.common.tenancy.__init__` developer primer.
- **`ensure_same_org` accepts both `TenantOwnedModel` subclasses and
  `Organization` instances.** Convenience for service-layer code that
  mixes the two (e.g. "ensure this Quote belongs to this Org").
  None values are skipped — convenience for optional FKs.
- **B.1.7 tests test more than the literal guide says** (PROTECT
  on_delete, canonical FK target). The guide gives a minimum; we
  enforce the maximum reasonable invariant. If any of these tighten
  in a future amendment, our tests already pass; if they loosen, we
  relax the test in the same PR as the guide amendment.
